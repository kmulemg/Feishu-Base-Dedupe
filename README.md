# 索引列重复检测与合并插件

## 插件简介

这是一个多维表格插件，用于检测可选字段是否存在重复数据，并在发现重复时提供智能合并方案。

## 主要功能

### 核心功能
1. **自动字段选择**：打开插件后自动读取当前表格字段，默认选择第一列作为检测字段
2. **灵活字段切换**：用户可手动切换检测字段
3. **重复数据检测**：点击"开始检测"后，读取当前表格记录，将所选字段值相同的记录归为重复组
4. **智能冲突分析**：自动识别字段冲突，区分可自动合并字段和需要手动选择的冲突字段

### 合并策略
- **主记录选择**：自动选择字段数据最完整的记录作为主记录
- **自动合并**：对于只有一个有效值或所有值相同的字段，自动写入主记录
- **手动选择**：对于存在多个不同有效值的冲突字段，弹出"字段冲突对比与选择"窗口让用户选择保留值
- **批量处理**：确认后更新主记录，并删除同组其他重复记录

### 界面特性
- 支持 Light/Dark 主题自动切换
- 国际化支持（中文/英文）
- 响应式设计，适配不同屏幕尺寸

## 部署到多维表格

### 本地开发模式

1. 启动开发服务器：`npm run dev`
2. 打开飞书多维表格
3. 点击「插件」→「自定义插件」→「新增插件」
4. 输入本地服务地址（如：`http://localhost:5173`）
5. 点击「确定」添加并运行插件

### 部署到云端

推荐使用以下平台部署：

- **Vercel**：推荐，支持一键部署 GitHub 仓库
- **Netlify**：免费额度充足，适合静态站点
- **GitHub Pages**：免费，但需要配置自定义域名
- **Replit**：在线开发环境，直接导入使用

部署后，将生成的 URL 填入多维表格插件地址即可。

## 核心组件说明

### App.tsx
主应用组件，包含：
- 字段选择逻辑
- 重复数据检测算法
- 智能冲突分析
- 记录合并执行

### ConflictDialog.tsx
冲突策略选择对话框，提供三种合并策略：
- **覆盖**：使用最后一条记录的数据覆盖主记录
- **保留**：保持主记录不变，跳过合并
- **合并**：遍历待合并记录，填充主记录的空字段

### FieldConflictDialog.tsx
字段冲突对比选择对话框：
- 展示所有冲突字段的详细对比信息
- 允许用户手动选择需要保留的字段值
- 显示冲突数量统计

### useTheme.ts
主题管理 Hook：
- 自动获取当前主题模式
- 监听主题变化
- 动态应用主题样式

## 注意事项

1. **数据安全**：插件仅在本地处理数据，不会向外部服务器发送任何信息
2. **批量操作**：对于大量数据，合并操作可能需要几秒钟时间
3. **权限要求**：插件需要读取和写入多维表格数据的权限
4. **备份建议**：在进行大规模数据合并前，建议备份原始数据
5. **只读字段**：公式、创建时间、修改时间、创建人、修改人、自动编号等字段无法通过 API 修改，会被自动跳过

## 常见问题

### Q: 合并操作可以撤销吗？

A: 当前版本不支持撤销操作，请在执行合并前确认数据。建议在合并前备份重要数据。

### Q: 支持同时处理多个重复记录吗？

A: 是的，插件会自动检测所有重复的字段，并在合并时统一处理。

### Q: 插件支持哪些字段类型？

A: 插件支持大部分字段类型，但以下只读字段无法修改：
- 查找引用（Lookup）
- 公式（Formula）
- 创建时间（CreatedTime）
- 修改时间（ModifiedTime）
- 创建人（CreatedUser）
- 修改人（ModifiedUser）
- 自动编号（AutoNumber）
